# Clipdrop Cleanup и удаление фона — дизайн

## Цель

Добавить в Manga Studio два AI-действия для текущего изображения:

1. **Удалить объект** — удалить отмеченную пользователем область через Clipdrop Cleanup API.
2. **Удалить фон** — автоматически получить изображение с прозрачным фоном через Clipdrop Remove Background API.

Оба действия должны быть защищены серверными маршрутами, не раскрывать API-ключ браузеру, сохранять исходник и отменяться одним шагом Undo/Redo.

## Границы первой версии

Входит:

- обработка только активного изображения;
- ручная чёрно-белая маска для удаления объектов;
- автоматическое удаление фона без маски;
- прогресс, отмена запроса, понятные ошибки и защита от повторного запуска;
- применение результата в существующий committed cleanup-слой;
- один атомарный шаг истории на успешную операцию;
- сохранение прозрачности PNG при удалении фона;
- серверная валидация входных файлов и ответа Clipdrop.

Не входит:

- пакетная обработка всех страниц;
- хранение результатов на сервере;
- очереди, фоновые задания и повторные попытки после перезагрузки;
- новый универсальный растровый layer engine;
- автоматическая генерация маски для Cleanup;
- сохранение API-ключа в коде, Git или клиентском состоянии.

## Архитектура

### Серверные маршруты

Используются два отдельных Next.js Route Handler:

- `POST /api/clipdrop/cleanup`
- `POST /api/clipdrop/remove-background`

Оба маршрута работают в Node.js runtime и читают `CLIPDROP_API_KEY` только на сервере.

`/cleanup` принимает `multipart/form-data`:

- `image_file`: PNG/JPEG/WebP-композит текущего изображения;
- `mask_file`: чёрно-белый PNG того же пиксельного размера.

`/remove-background` принимает:

- `image_file`: PNG/JPEG/WebP-композит текущего изображения.

Маршруты пересылают запрос в соответствующий Clipdrop endpoint, проверяют HTTP-статус, `Content-Type`, непустое тело и лимиты размера, затем возвращают бинарный результат браузеру. Ответы не кэшируются.

### Клиентский сервис

Отдельный модуль инкапсулирует:

- создание `FormData`;
- `fetch` к внутренним маршрутам;
- `AbortSignal`;
- преобразование бинарного ответа в Blob/data URL;
- нормализацию ошибок в пользовательские русские сообщения.

Компоненты панели не знают формат Clipdrop-запросов.

### Состояние редактора

Результат записывается в существующий `cleanup.committed`. Исходный документ не изменяется. Store получает отдельное атомарное действие применения результата, которое один раз вызывает существующий механизм истории. Служебные mask-штрихи после успешного Cleanup удаляются внутри того же изменения; при Undo восстанавливаются committed-слой и состояние до операции.

Удаление фона возвращает PNG с alpha-каналом. Canvas и экспорт обязаны отрисовывать committed PNG без белой подложки поверх прозрачных областей.

## Потоки данных

### Удаление объекта

1. Пользователь выбирает «Удалить объект».
2. Редактор активирует `maskBrush`; полупрозрачная оранжевая маска видна только в UI.
3. Кнопка запуска доступна только при наличии валидной маски.
4. Клиент создаёт растровый композит без UI, рамок выделения и служебной маски.
5. Маска растрируется в PNG исходного размера: белый — удалить, чёрный — сохранить.
6. Клиент отправляет оба Blob внутреннему `/api/clipdrop/cleanup`.
7. Успешный PNG применяется атомарно к active document; маска очищается.
8. Ошибка сохраняет маску, чтобы пользователь мог повторить запрос.

### Удаление фона

1. Пользователь нажимает «Удалить фон».
2. Клиент создаёт актуальный композит без UI и служебной маски.
3. Blob отправляется в `/api/clipdrop/remove-background`.
4. PNG с прозрачностью применяется одним шагом истории.
5. Наличие mask-штрихов не влияет на запрос и не очищается.

## Интерфейс

В существующей панели «Очистка» добавляется секция **Clipdrop AI** без изменения общей компоновки редактора.

Элементы:

- режим/действие «Удалить объект»;
- подсказка «Закрасьте область, которую нужно удалить»;
- кнопка «Очистить маску»;
- primary-кнопка «Удалить отмеченное»;
- отдельная кнопка «Удалить фон»;
- компактный индикатор обработки;
- кнопка «Отменить запрос» во время выполнения.

Во время запроса конфликтующие кнопки отключены. UI использует существующие design tokens, синий accent, нейтральные панели и текущую типографику; новые цвета не вводятся, кроме уже используемого оранжевого overlay маски.

## Валидация и безопасность

- API-ключ хранится только в `CLIPDROP_API_KEY`.
- Ранее опубликованный ключ считается скомпрометированным и должен быть отозван.
- Разрешены только ожидаемые MIME-типы изображений.
- Изображение и маска должны иметь одинаковые размеры до отправки.
- Пустая маска отклоняется на клиенте и сервере.
- Размер входа ограничивается константой маршрута; слишком большие файлы получают `413`.
- Ошибки Clipdrop не возвращают секреты или сырые внутренние данные.
- Маршруты используют `Cache-Control: no-store`.
- Клиент не логирует изображения, ключи или бинарные ответы.

## Ошибки

Пользовательские состояния:

- ключ не настроен;
- маска пуста;
- изображение слишком большое;
- формат не поддерживается;
- лимит/кредиты Clipdrop исчерпаны;
- Clipdrop временно недоступен;
- запрос отменён;
- ответ сервиса повреждён.

При любой ошибке документ и история не меняются. Для Cleanup нарисованная маска сохраняется.

## Тестирование

### Unit

- растеризация маски создаёт PNG нужного размера и правильной полярности;
- пустая маска определяется корректно;
- нормализация Clipdrop-ошибок;
- атомарное применение результата создаёт ровно один history entry;
- Undo/Redo восстанавливает результат и mask-штрихи.

### Route handlers

- отсутствующий ключ → контролируемая ошибка;
- неправильный MIME/размер → `400`/`413`;
- upstream `401`, `402/429`, `5xx` → безопасные нормализованные ответы;
- успешный бинарный PNG проходит без изменения;
- `Cache-Control: no-store` присутствует.

### Browser regression

- кисть маски → удалить объект → Undo → Redo;
- ошибка API не стирает маску;
- удалить фон → прозрачность видна на canvas и сохраняется в PNG/ZIP;
- переключение между 10+ изображениями не переносит результат на другую страницу;
- повторный клик во время запроса не создаёт дубликат;
- окно около 1024 px сохраняет доступ к обеим кнопкам;
- экспорт имеет исходный размер изображения.

## Критерии готовности

- ключ Clipdrop отсутствует в клиентском bundle и Git diff;
- оба режима работают через отдельные серверные маршруты;
- Cleanup использует маску исходного размера;
- Remove Background сохраняет alpha-канал;
- успешная операция — один шаг Undo/Redo;
- неуспешная операция не меняет документ;
- экспорт совпадает с canvas и сохраняет оригинальные размеры;
- TypeScript, production build и browser regression проходят.
